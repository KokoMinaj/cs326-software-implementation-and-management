# QA Plan

## Test Levels
- Unit Testing – tests individual functions/components
- Integration Testing – tests interaction between modules
- System Testing – tests the entire application

## Entry Criteria
- Code is complete
- No syntax errors
- Dependencies installed

## Exit Criteria
- All tests pass
- No critical bugs (S1/S2)
- Code reviewed and approved

## Severity Levels
- S1 – Critical (system crash, data loss)
- S2 – Major (feature broken)
- S3 – Minor (small issue, workaround exists)
- S4 – Low (UI/typo issues)
