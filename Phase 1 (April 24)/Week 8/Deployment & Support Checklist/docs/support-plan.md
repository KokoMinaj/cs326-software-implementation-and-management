# Support Plan - QueueLess Backend

This document defines the procedures for handling issues and supporting users of the QueueLess system.

## 1. Issue Reporting Process
Users and stakeholders should report defects or outages via the following channels:
- **GitHub Issues**: Preferred for technical bugs and feature requests.
- **Support Email**: coding.cyrilj777@gmail.com for account or urgent access issues.
- **Emergency Slack/Discord**: For critical production outages (Internal Team only).

## 2. Issue Categorization
| Severity | Description | Example |
|----------|-------------|---------|
| **P0 (Critical)** | System is down; users cannot join queues. | Server 500 error on all requests. |
| **P1 (High)** | Major feature broken; notifications not sending. | SMS alerts failing for all users. |
| **P2 (Normal)** | Minor bug; UI/UX inconsistency. | Typo in notification message. |
| **P3 (Low)** | General question or feature request. | Requesting a new institution type. |

## 3. Response Times (SLA)
We aim to acknowledge and respond to issues within these timeframes:
- **P0**: 2 hours (24/7 coverage)
- **P1**: 4 hours (Business hours)
- **P2**: 24 hours
- **P3**: 3-5 business days

## 4. Maintenance Window
Scheduled maintenance occurs every **Sunday at 02:00 AM UTC**. Users will see a "System Maintenance" banner 1 hour prior to the window.
